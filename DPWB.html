CAPÍTULO 1
HTML — Los Ladrillos de la Web

🌐 ¿Qué es HTML? Lo más importante

HTML significa HyperText Markup Language (Lenguaje de Marcado de Hipertexto). Es el lenguaje que usamos para decirle al navegador (Chrome, Firefox, etc.) qué cosas debe mostrar en una página web.

HTML no es un lenguaje de programación como los que hacen videojuegos. Es un lenguaje de estructura: le dice al navegador si algo es un título, un párrafo, una imagen o un botón.

🏠
Imagínalo así: Si una página web fuera una casa, el HTML sería la estructura — las paredes, el techo y las ventanas. Sin HTML, no hay casa.
¿Por qué existe HTML? Porque las computadoras necesitan instrucciones muy claras. Si escribes "Hola" en un archivo de texto, la computadora no sabe si es un título gigante o un párrafo pequeño. HTML le da ese contexto.

💻 ¿Qué es Visual Studio Code?

Visual Studio Code (VS Code) es un programa gratuito creado por Microsoft que usamos para escribir código. Es como un cuaderno especial para programadores.

Cuando escribes código en VS Code, el programa hace cosas geniales automáticamente:

1

Colores en el código — Pinta cada parte de un color diferente para que sea fácil de leer.

2

Autocompletado — Si empiezas a escribir una etiqueta, VS Code la termina por ti.

3

Detecta errores — Si algo está mal escrito, te avisa con una línea roja.

¿Por qué usarlo? Podrías escribir HTML en el Bloc de notas, pero VS Code hace todo mucho más fácil. Es como escribir con lápiz normal vs. escribir con un lápiz mágico que corrige solo.
🎉
Dato curioso: VS Code es completamente gratis y más de 14 millones de programadores en el mundo lo usan todos los días.

🏗️ Estructura Básica de HTML

Todo archivo HTML empieza igual. Tiene unas etiquetas principales que siempre deben estar. Mira el esqueleto:

<!DOCTYPE html>  <!-- Le dice al navegador que esto es HTML5 -->
<html lang="es">    <!-- Todo el HTML va aquí dentro -->

  <head>               <!-- Información que el visitante NO ve -->
    <meta charset="UTF-8">
    <title>Mi primera página</title>
  </head>

  <body>               <!-- Todo lo que el visitante SÍ ve -->
    <h1>¡Hola, Mundo!</h1>
    <p>Esta es mi primera página web.</p>
  </body>

</html>
Etiqueta¿Para qué sirve?
<!DOCTYPE html>Avisa que usamos la versión más nueva de HTML
<html>El contenedor de toda la página
<head>Información técnica (título, fuentes, etc.) — no se ve en pantalla
<title>El texto que aparece en la pestaña del navegador
<body>Todo lo que el usuario ve en pantalla
¿Por qué siempre la misma estructura? Es como las reglas de un examen: siempre escribes tu nombre arriba, la fecha, el título. El navegador necesita ese orden para entender dónde empieza y termina cada sección.

🏷️ Etiquetas Comunes

Una etiqueta es como una orden entre corchetes angulares < >. La mayoría tienen una etiqueta de apertura y una de cierre (con /).

<h1> a <h6> <p> <a> <img> <ul> <li> <div> <span> <strong> <button> <form> 
<!-- Títulos: h1 es el más grande, h6 el más pequeño -->
<h1>Título Principal</h1>
<h2>Subtítulo</h2>

<!-- Párrafo de texto -->
<p>Soy un párrafo de texto normal.</p>

<!-- Enlace (link) -->
<a href="https://google.com">Ir a Google</a>

<!-- Imagen -->
<img src="gato.jpg" alt="Un gato bonito">

<!-- Lista con puntos -->
<ul>
  <li>Manzana</li>
  <li>Banana</li>
</ul>
¿Por qué hay tantas etiquetas? Porque cada tipo de contenido es diferente. Un título no es lo mismo que un párrafo, y un enlace no es lo mismo que una imagen. Tener etiquetas distintas le ayuda al navegador (y a Google) a entender bien tu página.

🎵 Agregar Audio

Para poner un sonido o canción en tu página web usamos la etiqueta <audio>.

<audio controls>
  <source src="cancion.mp3" type="audio/mpeg">
  Tu navegador no soporta audio.
</audio>
Atributo¿Qué hace?
controlsMuestra botones de play, pausa y volumen
autoplayEmpieza a sonar automáticamente (¡úsalo con cuidado!)
loopRepite el audio una y otra vez
mutedEmpieza en silencio
¿Por qué usamos <source> adentro? Porque no todos los navegadores entienden los mismos formatos de audio. Puedes poner varios <source> y el navegador elegirá el que pueda reproducir.

🎬 Agregar Video

Para poner un video en tu página usamos la etiqueta <video>. Funciona muy parecido al audio.

<video controls width="640" height="360">
  <source src="pelicula.mp4" type="video/mp4">
  Tu navegador no soporta video.
</video>

<!-- También puedes insertar un video de YouTube con iframe -->
<iframe
  src="https://www.youtube.com/embed/ID_DEL_VIDEO"
  width="640" height="360"
  allowfullscreen>
</iframe>
📺
Analogía: El <video> es como poner un televisor dentro de tu página web. Tú decides el tamaño del televisor con width y height.

🧠 HTML Semántico Nivel Pro

Semántico significa "con significado". El HTML semántico usa etiquetas cuyos nombres describen exactamente para qué sirven, en vez de usar <div> para todo.

<!-- ❌ Sin semántica (difícil de entender) -->
<div id="encabezado"> ... </div>
<div id="nav"> ... </div>
<div id="contenido"> ... </div>

<!-- ✅ Con semántica (fácil de entender) -->
<header>  <!-- Encabezado de la página -->
  <nav>   <!-- Menú de navegación -->
    <a href="/">Inicio</a>
  </nav>
</header>

<main>    <!-- Contenido principal -->
  <article><!-- Un artículo o publicación -->
    <h1>Mi historia</h1>
    <p>Había una vez...</p>
  </article>
  <aside>  <!-- Información secundaria / barra lateral -->
    <p>Artículos relacionados...</p>
  </aside>
</main>

<footer>  <!-- Pie de página -->
  <p>© 2025 Mi Página</p>
</footer>
¿Por qué usar HTML semántico? Tres razones: (1) Google entiende mejor tu página y la recomienda más. (2) Las personas ciegas que usan lectores de pantalla pueden navegar mejor. (3) Tú mismo (o tus amigos) pueden entender el código más fácilmente.

CSS — El Maquillaje de la Web

🎨 ¿Qué es CSS?

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada). Es el lenguaje que usamos para darle apariencia a una página web: colores, tamaños, fuentes, espacios y mucho más.

Si HTML es la estructura de la casa, entonces CSS es la pintura, los muebles y la decoración.

🎭
Imagínalo así: HTML construye los actores del teatro (el título, el párrafo, la imagen). CSS les pone el disfraz: color de ropa, tamaño, posición en el escenario.
¿Por qué está separado del HTML? Para mantener las cosas ordenadas. El HTML dice qué hay en la página. El CSS dice cómo se ve. Si los mezclas todo, se vuelve un desastre muy difícil de cambiar.

⚙️ ¿Cómo Funciona CSS?

CSS funciona con reglas. Cada regla tiene dos partes: a quién le aplica (selector) y qué cambio hacer (propiedad y valor).

/* Estructura de una regla CSS */

selector {
  propiedad: valor;
  propiedad: valor;
}

/* Ejemplo real */
p {
  color:       blue;        /* texto azul */
  font-size:   18px;       /* letra de 18 píxeles */
  font-family: Arial;      /* tipo de letra Arial */
  background:  yellow;     /* fondo amarillo */
}

La palabra "Cascading" (en cascada) significa que si dos reglas intentan cambiar lo mismo, gana la que está más abajo en el archivo, como el agua que cae y la última en llegar llena el vaso.

📄 CSS con la etiqueta <style>

Una forma de agregar CSS es escribirlo dentro del mismo archivo HTML, usando la etiqueta <style> en el <head>.

<!DOCTYPE html>
<html>
<head>
  <title>Mi página</title>

  <style>
    h1 {
      color: red;
      text-align: center;
    }
    p {
      font-size: 16px;
      color: #333333;
    }
  </style>

</head>
<body>
  <h1>¡Hola!</h1>
  <p>Este texto tiene estilo.</p>
</body>
</html>
¿Cuándo usar <style>? Es bueno cuando estás aprendiendo o tienes una página pequeña. Para proyectos grandes es mejor usar un archivo separado (lo veremos enseguida).

🔗 CSS desde un Archivo Externo

La forma más profesional es guardar el CSS en un archivo separado con extensión .css y conectarlo al HTML con la etiqueta <link>.

Así quedarían tus archivos:

📁 Mi Proyecto/
├── index.html
└── estilos.css

Dentro de index.html, en el <head>:

<link rel="stylesheet" href="estilos.css">

Dentro de estilos.css:

body {
  background-color: #f0f0f0;
  font-family: Arial, sans-serif;
}

h1 {
  color: #264de4;
}
¿Por qué es mejor el archivo externo? Porque si tienes 50 páginas web y quieres cambiar el color del título, solo cambias un archivo y se actualiza en todas las páginas. ¡Sin esto tendrías que cambiar los 50 archivos uno por uno!

🎯 Selectores: Tipo, ID y Clase

Un selector le dice al CSS a cuál elemento HTML quiere cambiar. Hay tres tipos principales:

1. Selector de Tipo 🏷️

Afecta a todas las etiquetas de ese tipo.

p { color: green; }
h1 { color: red; }

👉 Pinta todos los párrafos de verde.

2. Selector de ID 🆔

Afecta a un solo elemento único. Se pone con #.

/* En el HTML */
<h1 id="titulo-rojo">Hola</h1>

/* En el CSS */
#titulo-rojo { color: red; }

👉 Solo afecta a ese h1 específico.

3. Selector de Clase 🏫

Afecta a varios elementos que compartan la misma clase. Se pone con . (punto).

/* En el HTML puedes usarlo en varios elementos */
<p class="importante">Este es importante</p>
<h2 class="importante">Este también</h2>
<span class="importante">Y este</span>

/* En el CSS (con punto al inicio) */
.importante {
  color: purple;
  font-weight: bold;
}

👉 Los tres elementos se volverán morados y negritas. ¡Muy útil para reutilizar estilos!

TipoSintaxis HTMLSintaxis CSS¿Cuántos afecta?
Tipo<p>p { }Todos los <p>
IDid="nombre"#nombre { }Solo 1 (único)
Claseclass="nombre".nombre { }Todos con esa clase
¿Cuándo uso cada uno? Usa tipo para reglas generales (todos los párrafos iguales), ID para un elemento único (solo ese logo, solo ese botón principal) y clase para grupos de elementos que deben verse igual (todos los botones rojos, todas las tarjetas con sombra).
